## Debanize pkg103greet
Here the objective is to debanize the library.

### Go version
Actual host is Debian stretch, but with stretch-backport. 
```bash
go version
go version go1.8.1 linux/amd64
```

### Use dh-make-golang
Set environment
```bash
export GOPATH=$HOME/go
export GOBIN=$GOPATH/bin
export PATH=$GOBIN
```
Use option type *program|library* since dh-make-golang can easily be confused by the files in the repository.
```bash
mkdir -p ~/PKGDEB/pkg103greet
cd ~/PKGDEB/PKG103GREET
dh-make-golang -type library github.com/berrak/pkg103greet
```
Directory (golang-github-berrak-pkg103greet) and files after above run:
```bash
golang-github-berrak-pkg103greet
golang-github-berrak-pkg103greet_0.0~git20180303.ed1dabe.orig.tar.xz
golang-github-berrak-pkg103greet.txt

```
Library names are mangeled to package names by Debian, here as *golang-github-berrak-pkg103greet*.
Ignore the text file, its only for Debian internal usage. Lets look into **golang-github-berrak-pkg103greet** directory.
It contains three git branches:
```bash
git vbr
* master
  pristine-tar
  upstream
```
Switch to pristine-tar branch and confirm have same base name as the original tarball:
```bash
git sw2 pristine-tar
ls -l
golang-github-berrak-pkg103greet_0.0~git20180303.ed1dabe.orig.tar.xz.delta
golang-github-berrak-pkg103greet_0.0~git20180303.ed1dabe.orig.tar.xz.id

git sw2 master
```
Edit **changelog** in the debian directory.
```bash
cd debian
vi changelog
```
Replace *UNRELEASED* with stable and remove TODO with (Closes: #123456) or whatever.
Next view the file **control**, but initially it is acceptable for the first packaging attempt.
Edit **rules** and add DH_GOPKG stanza as below:
```bash
#!/usr/bin/make -f

export DH_GOPKG := github.com/berrak/pkg103greet

%:
        dh $@ --buildsystem=golang --with=golang    
```

Add all debian files to git:
```bash
cd debian
git add .
git com -m 'Initial packaging'
```
Now we can debanize the *golang-github-berrak-pkg103greet* library in the chrooted stretch (note: golang-1.7-go (1.7.4-2)) environment.
